# BioHacker_Editor

一个用于记录和管理健康日志的 Web 应用，数据存储在 GitHub 上。

## 项目定位

BioHacker_Editor 是一个简洁、美观的健康日志管理工具，帮助你记录每日的健康数据，包括天气、睡眠、饮食、运动等信息，并通过 GitHub 进行数据持久化存储。

## 基本功能

- 📅 **每日记录**：创建和管理每日健康日志卡片
- 🌤️ **天气查询**：自动查询并填充当地天气信息
- 😴 **睡眠记录**：记录入睡时间、醒来时间和睡眠质量
- 🍽️ **饮食追踪**：记录早午晚三餐
- 🏃 **运动记录**：记录每日运动情况
- 💡 **特殊标注**：支持 [+] 鼓励、[?] 观察、[!] 问题三种标注
- 🔄 **分页浏览**：方便查看历史记录
- 💻 **控制台命令**：支持快速创建和跳转到指定日期的记录

## 设置配置项填写教程

### 1. 打开设置

点击右上角的「设置」按钮，打开设置面板。

### 2. 配置项说明

#### GitHub Token
- **用途**：用于访问 GitHub API，读取和写入你的健康日志文件
- **获取方式**：
  1. 访问 GitHub Settings → Developer settings → Personal access tokens
  2. 点击「Generate new token」→ 选择「Generate new token (classic)」
  3. 勾选 `repo` 权限
  4. 生成并复制 Token（注意：Token 只显示一次，请妥善保存）
- **格式示例**：`ghp_xxxxxxxxxxxxxxxxxxxx`

#### 用户名
- **用途**：你的 GitHub 用户名
- **格式示例**：`your-username`

#### 仓库名
- **用途**：存储健康日志的 GitHub 仓库名称
- **注意事项**：需要先在 GitHub 上创建一个仓库
- **格式示例**：`health-log`

#### 文件路径
- **用途**：仓库中存储健康日志的文件路径
- **格式示例**：`health.txt`

#### 所在城市
- **用途**：用于查询当地天气
- **格式示例**：`深圳` 或 `南宁 青秀区`

### 3. 保存设置

填写完所有配置项后，点击「保存设置」按钮，设置会自动保存到浏览器本地存储中。

### 4. 加载数据

设置完成后，点击「刷新」按钮即可从 GitHub 加载你的健康日志数据。

## 控制台命令

### 触发方式
- **桌面端**：依次按下 `C` → `M` → `D` 键
- **移动端**：4秒内连续点击4次左上角的创建卡片按钮

### 命令列表
- `create(yyyy.m.d)` - 创建指定日期的卡片
- `tp(yyyy.m.d)` - 跳转到指定日期的卡片页面

详细命令文档请参考 [Function/console/README.md](Function/console/README.md)。

## 数据格式

健康日志以纯文本格式存储在 GitHub 上，格式如下：

```
2024.6.15
Weather: 晴，25-32℃
Sleep: 23:00 睡着，7:30 苏醒
Diet: 早饭：牛奶面包、午饭：米饭炒菜、晚饭：面条
Exercise: 跑步30分钟
Note: 今天状态不错
[+] 坚持运动
[?] 注意睡眠质量
```
